技术栈：PostgreSQL + pgvector


## 先决条件：启用pgvector扩展

```sql
-- 在数据库中创建扩展（只需执行一次）
CREATE EXTENSION IF NOT EXISTS vector;
```

## 数据库表设计

### 1. 用户表 (users)
存储学生基本信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 用户ID |
| username | VARCHAR(50) | UNIQUE NOT NULL | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | 加密密码 |
| phone | VARCHAR(20) | | 手机号 |
| email | VARCHAR(100) | | 邮箱 |
| real_name | VARCHAR(50) | | 真实姓名 |
| school | VARCHAR(100) | | 学校 |
| major | VARCHAR(100) | | 专业 |
| grade | VARCHAR(20) | | 年级（如“大三”） |
| default_job_id | INT | REFERENCES jobs(id) | 默认岗位ID |
| avatar_url | TEXT | | 头像URL |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | 注册时间 |

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    phone VARCHAR(20),
    email VARCHAR(100),
    real_name VARCHAR(50),
    school VARCHAR(100),
    major VARCHAR(100),
    grade VARCHAR(20),
    default_job_id INT REFERENCES jobs(id),
    avatar_url TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 2. 岗位表 (jobs)
支持的面试岗位。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 岗位ID |
| name | VARCHAR(50) | NOT NULL | 岗位名称，如“Java后端开发” |
| description | TEXT | | 岗位简介 |
| tech_stack | JSONB | | 核心技术栈（JSON数组） |
| icon_url | TEXT | | 图标URL |

```sql
CREATE TABLE jobs (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    description TEXT,
    tech_stack JSONB,
    icon_url TEXT
);
```

### 3. 面试记录表 (interviews)
每次面试的概要信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 面试ID |
| user_id | INT | NOT NULL REFERENCES users(id) ON DELETE CASCADE | 用户ID |
| job_id | INT | NOT NULL REFERENCES jobs(id) | 岗位ID |
| start_time | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | 开始时间 |
| end_time | TIMESTAMPTZ | | 结束时间 |
| total_score | INT | | 综合得分（百分制） |
| status | VARCHAR(20) | DEFAULT 'in_progress' | 状态：in_progress, completed |
| question_count | INT | DEFAULT 0 | 总题数 |
| used_time | INT | | 用时（秒） |

```sql
CREATE TABLE interviews (
    id SERIAL PRIMARY KEY,
    user_id INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    job_id INT NOT NULL REFERENCES jobs(id),
    start_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    end_time TIMESTAMPTZ,
    total_score INT CHECK (total_score >= 0 AND total_score <= 100),
    status VARCHAR(20) DEFAULT 'in_progress',
    question_count INT DEFAULT 0,
    used_time INT
);
CREATE INDEX idx_interviews_user_id ON interviews(user_id);
```

### 4. 面试对话表 (interview_chats)
记录面试中的每一轮对话。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 对话ID |
| interview_id | INT | NOT NULL REFERENCES interviews(id) ON DELETE CASCADE | 所属面试ID |
| role | VARCHAR(10) | NOT NULL CHECK (role IN ('ai', 'user')) | 发言角色 |
| content | TEXT | NOT NULL | 内容 |
| timestamp | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | 时间戳 |
| duration | INT | | 用户回答耗时（秒），仅当role='user'时 |
| question_id | INT | REFERENCES questions(id) | 对应题库问题ID（如果可关联） |

```sql
CREATE TABLE interview_chats (
    id BIGSERIAL PRIMARY KEY,
    interview_id INT NOT NULL REFERENCES interviews(id) ON DELETE CASCADE,
    role VARCHAR(10) NOT NULL CHECK (role IN ('ai', 'user')),
    content TEXT NOT NULL,
    timestamp TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    duration INT,
    question_id INT REFERENCES questions(id)
);
CREATE INDEX idx_chats_interview_id ON interview_chats(interview_id);
```

### 5. 评分维度表 (dimensions)
面试评分维度。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 维度ID |
| name | VARCHAR(50) | UNIQUE NOT NULL | 维度名称，如“技术正确性” |
| description | TEXT | | 描述 |

```sql
CREATE TABLE dimensions (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT
);
-- 初始化五个维度
INSERT INTO dimensions (name) VALUES
('技术正确性'), ('逻辑严谨性'), ('岗位匹配度'), ('表达沟通'), ('应变能力');
```

### 6. 面试评分表 (interview_scores)
每次面试在各维度的得分。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 评分ID |
| interview_id | INT | NOT NULL REFERENCES interviews(id) ON DELETE CASCADE | 面试ID |
| dimension_id | INT | NOT NULL REFERENCES dimensions(id) | 维度ID |
| score | INT | NOT NULL CHECK (score >= 0 AND score <= 100) | 得分 |
| comment | TEXT | | 评语 |

```sql
CREATE TABLE interview_scores (
    id SERIAL PRIMARY KEY,
    interview_id INT NOT NULL REFERENCES interviews(id) ON DELETE CASCADE,
    dimension_id INT NOT NULL REFERENCES dimensions(id),
    score INT NOT NULL CHECK (score >= 0 AND score <= 100),
    comment TEXT,
    UNIQUE (interview_id, dimension_id)
);
CREATE INDEX idx_scores_interview_id ON interview_scores(interview_id);
```

### 7. 面试题表 (questions) —— 题库（带向量）
存储面试题目，用于AI提问和检索。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 题目ID |
| job_id | INT | NOT NULL REFERENCES jobs(id) | 所属岗位 |
| content | TEXT | NOT NULL | 问题内容 |
| type | VARCHAR(20) | NOT NULL CHECK (type IN ('basic', 'scenario', 'followup')) | 问题类型：基础/场景/追问 |
| difficulty | VARCHAR(10) | CHECK (difficulty IN ('easy','medium','hard')) | 难度 |
| keywords | JSONB | | 关键词数组 |
| reference_answer | TEXT | | 参考答案要点 |
| knowledge_points | JSONB | | 关联知识点标签数组 |
| embedding | VECTOR(1536) | | 问题内容的向量（使用1536维，可根据模型调整） |

```sql
CREATE TABLE questions (
    id SERIAL PRIMARY KEY,
    job_id INT NOT NULL REFERENCES jobs(id),
    content TEXT NOT NULL,
    type VARCHAR(20) NOT NULL CHECK (type IN ('basic', 'scenario', 'followup')),
    difficulty VARCHAR(10) CHECK (difficulty IN ('easy','medium','hard')),
    keywords JSONB,
    reference_answer TEXT,
    knowledge_points JSONB,
    embedding VECTOR(1536)
);
CREATE INDEX idx_questions_job_id ON questions(job_id);
-- 向量索引（数据插入后创建）
CREATE INDEX idx_questions_embedding ON questions USING ivfflat (embedding vector_cosine_ops);
```

### 8. 学习资源表 (resources)
存储推荐的学习资料（文章、视频等），带向量。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 资源ID |
| title | VARCHAR(200) | NOT NULL | 标题 |
| type | VARCHAR(20) | NOT NULL CHECK (type IN ('article','video','course','example')) | 资源类型 |
| url | TEXT | | 链接 |
| content | TEXT | | 全文或摘要（用于向量化） |
| source | VARCHAR(100) | | 来源 |
| thumbnail | TEXT | | 缩略图 |
| difficulty | VARCHAR(10) | CHECK (difficulty IN ('easy','medium','hard')) | 难度 |
| tags | JSONB | | 标签数组 |
| embedding | VECTOR(1536) | | 内容向量 |

```sql
CREATE TABLE resources (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    type VARCHAR(20) NOT NULL CHECK (type IN ('article','video','course','example')),
    url TEXT,
    content TEXT,
    source VARCHAR(100),
    thumbnail TEXT,
    difficulty VARCHAR(10) CHECK (difficulty IN ('easy','medium','hard')),
    tags JSONB,
    embedding VECTOR(1536)
);
CREATE INDEX idx_resources_embedding ON resources USING ivfflat (embedding vector_cosine_ops);
```

### 10. 资源-知识点关联表 (resource_tags)
多对多关联资源与知识点。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| resource_id | INT | REFERENCES resources(id) ON DELETE CASCADE | 资源ID |

| PRIMARY KEY (resource_id, tag_id) | | | 联合主键 |

```sql
CREATE TABLE resource_tags (
    resource_id INT REFERENCES resources(id) ON DELETE CASCADE,

```

### 14. 提示词/角色配置表 (ai_prompts)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | |
| name | VARCHAR(100) | NOT NULL | 配置名称，如“Java后端面试官-正式” |
| job_id | INT | REFERENCES jobs(id) ON DELETE SET NULL | 关联的岗位（NULL表示通用） |
| role_description | TEXT | | 角色描述，如“你是一名资深Java技术专家，面试风格严谨” |
| system_prompt | TEXT | | 系统级提示词（给AI的初始指令） |
| greeting_message | TEXT | | 开场白模板 |
| questioning_style | VARCHAR(50) | | 提问风格：友好/严格/引导式等 |
| temperature | DECIMAL(2,1) | DEFAULT 0.7 | 模型温度参数 |
| max_tokens | INT | DEFAULT 500 | 最大输出token数 |
| is_active | BOOLEAN | DEFAULT true | 是否启用 |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | |

```sql
CREATE TABLE ai_prompts (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    job_id INT REFERENCES jobs(id) ON DELETE SET NULL,
    role_description TEXT,
    system_prompt TEXT,
    greeting_message TEXT,
    questioning_style VARCHAR(50),
    temperature DECIMAL(2,1) DEFAULT 0.7,
    max_tokens INT DEFAULT 500,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**作用**：
- 系统可以根据用户选择的岗位，自动加载对应的提示词配置，让AI面试官表现出符合该岗位特点的风格。
- 如果未来需要A/B测试不同风格的面试官，可以轻松增加新配置。

### 16. 通用面试知识库（ knowledge_items 表）
这部分存储的是**不直接作为题目，但对AI生成反馈和评分非常有帮助的知识**，例如：
- 面试技巧（如“如何结构化回答”）
- 常见错误（如“回答缺乏逻辑的典型表现”）
- 岗位能力模型（如Java后端需要掌握哪些技术栈及深度）
- 优秀回答示例（可以是对某个问题的满分回答文本）
- 行业最佳实践

这些知识可以让AI在评分和改进建议时更有依据。

#### 知识条目表 (knowledge_items)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | |
| title | VARCHAR(200) | | 标题 |
| content | TEXT | NOT NULL | 知识内容 |
| type | VARCHAR(50) | NOT NULL | 类型：tip / mistake / model / practice 等 |
| job_id | INT | REFERENCES jobs(id) | 关联岗位（NULL表示通用） |
| tags | JSONB | | 标签 |
| embedding | VECTOR(1536) | | 内容向量 |

```sql
CREATE TABLE knowledge_items (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200),
    content TEXT NOT NULL,
    type VARCHAR(50) NOT NULL,
    job_id INT REFERENCES jobs(id),
    tags JSONB,
    embedding VECTOR(1536)
);
CREATE INDEX idx_knowledge_embedding ON knowledge_items USING ivfflat (embedding vector_cosine_ops);
```

**使用场景举例**：
- 当AI需要给用户提出改进建议时，可以检索与当前回答相关的“常见错误”知识条目，提取其中的要点来生成个性化建议。
- 当用户询问“如何提高逻辑性”时，AI可以检索“面试技巧”类知识，并结合用户的具体情况回答。
